# Prompts Appendix 

This document details the exact primpts used in interactive Dual-LLM framework for extracting interpretable rules in power systems. The architecture aims to distill predictive capabilities of a Histogram Gradient Boosting - HGB into transparent, binary `if/else` logic rules for power-grid failure prediction. 
The ecosystem consists of three prompts: **Generator**, **Critic**, and **Syntax Repair**.

## 1. Rule Generator

The Generator acts as the optimization engine. Its role is to iterate over the rules, proposing new splits in the feature space (new thresholds) based on historical analysis, statistical data, and deterministic self-diagnosis heuristics.

### Dictionary of Variables
* `{{ iteration }}`: The current iteration number within the optimization loop.
* `{{ line_name }}`: The unique identifier of the disconnected transmission line currently under analysis.
* `{{ alerts }}`: Dynamically generated safety warnings (e.g., `oscillation_note` or `stagnation_note`).
* `{{ seed_section }}`: Baseline rules computed purely from data distribution statistics prior to iteration 1.
* `{{ HGB_TARGET_* }}`: The benchmark performance metrics (Acc, F2, OVR, FA) of the HGB model that the surrogate rule must attempt to approximate.
* `{{ diagnosis_text }}`: A deterministic heuristic (Cases A, B, C, or D) enforced by the system to restrict the LLM's search space and guide boundary adjustments.
* `{{ feature_stats_text }}`: Statistical summary (minimum, mean, maximum) of the features within the current training window.
* `{{ anchors_text }}`: Class-separated median differences used to suggest highly discriminative thresholds.
* `{{ feedback_text_clipped }}`: The critique and instructions generated by the Critic agent in the previous iteration.
* `{{ prev_rule_section }}` / `{{ prev_metrics_section }}`: The code and evaluation metrics of the rule generated in the immediately preceding iteration.
* `{{ best_rule_section }}` / `{{ best_metrics_section }}`: The global optimum found so far (Best So Far).
* `{{ history_table_json }}`: A JSON-formatted table tracking the metrics of recent iterations for trend analysis.
* `{{ training_json }}`: A balanced sample of the training data in JSON format.
* `{{ worst_cases_json }}`: The hardest topological scenarios where the current rule fails (prioritizing False Negatives, followed by False Positives).
* `{{ feature_description_text }}`: A static dictionary detailing the physical and semantic meaning of each grid feature.

### The Prompt
```text
You are the generator LLM in a two-LLM iterative framework for power-grid failure prediction.

Your task: Generate ONE improved Python if/else rule to predict binary failure risk,
imitating a teacher HGB model.
{{ alerts }}{{ seed_section }}
## Context
- Disconnected line: {{ line_name }}
- Failures are rare (imbalanced dataset)
- Teacher HGB: Acc={{ HGB_TARGET_acc }} | F2={{ HGB_TARGET_f2 }} | OVR={{ HGB_TARGET_ovr }} | FA={{ HGB_TARGET_fa }}

## Scoring formula
  score = 0.10*Acc + 0.40*F2 + 0.35*(1-OVR) + 0.15*(1-FA)
  Penalties:
    F2 == 0              → score = -1.00  (rule never catches failures — forbidden)
    FA >= 0.90           → score = -0.90  (rule fires almost always — also forbidden)
    OVR > 0.50 AND F2>0  → additional -0.30 progressive penalty on top of base
  Priority: F2 (0.40) > OVR (0.35) > FA (0.15) > Acc (0.10)
  TARGET CORRIDOR: OVR ∈ [0.04, 0.15] AND FA ∈ [0.05, 0.25]

## MANDATORY self-diagnosis (do this BEFORE writing the rule)
{{ diagnosis_text }}

{{ anchors_text }}

{{ feedback_text_clipped }}

- 2–5 conditions total; start from suggested_threshold values
